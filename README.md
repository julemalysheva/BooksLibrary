**Название проекта:** Онлайн библиотека "BooksLibrary"

**Краткое описание проекта:**
Книжная библиотека "BooksLibrary" - это веб-приложение, разработанное для управления книжным ассортиментом, организации заказов книг, обмена сообщениями между пользователями и администраторами, а также написания и просмотра отзывов о книгах. Основная цель проекта - облегчить процесс управления книгами и заказами, предоставить пользователям возможность общения и оценки литературных произведений.

**Назначение проекта:**

- Управление книжным ассортиментом: добавление, удаление и изменение информации о книгах.
- Оформление заказов на книги: возможность пользователей заказывать и возвращать книги, а также продление срока аренды.
- Обмен сообщениями: общение пользователей с администраторами для получения поддержки и решения вопросов.
- Отзывы о книгах: написание и просмотр отзывов пользователей о книгах.

**Цели проекта:**

1. Упростить процесс управления книжным ассортиментом и заказами.
2. Предоставить пользователейм удобный интерфейс для общения с администраторами.
3. Собрать обратную связь о книгах с помощью системы отзывов.
4. Создать надежную и безопасную платформу для пользователей и администраторов.

**Структура пакетов**

*ru.malysheva.bookslibrary.config*
Пакет содержит классы, отвечающие за конфигурацию приложения, такие как настройки безопасности, настройки REST-контроллеров и т. д.

*ru.malysheva.bookslibrary.controller*
Здесь находятся контроллеры, которые обрабатывают HTTP-запросы от клиентов. Каждый контроллер отвечает за определенный ресурс или функциональность.

*ru.malysheva.bookslibrary.dao*
Пакет содержит интерфейсы репозиториев, которые обеспечивают доступ к данным в базе данных. Эти интерфейсы определяют методы для выполнения операций CRUD (создание, чтение, обновление, удаление) с соответствующими сущностями.

*ru.malysheva.bookslibrary.entity*
Здесь находятся классы-сущности, представляющие объекты, хранящиеся в базе данных. Каждый класс обычно соответствует таблице в базе данных.

*ru.malysheva.bookslibrary.requestmodels*
Пакет содержит классы, представляющие модели запросов от клиента. Эти классы используются для передачи данных от клиента к серверу.

*ru.malysheva.bookslibrary.responsemodels*
Здесь находятся классы, представляющие модели ответов сервера. Они используются для передачи данных от сервера к клиенту.

*ru.malysheva.bookslibrary.service*
Пакет содержит сервисы, которые реализуют бизнес-логику приложения. Сервисы взаимодействуют с репозиториями для выполнения операций над данными.

*ru.malysheva.bookslibrary.utils*
Здесь содержатся утилитарные классы, предоставляющие различные вспомогательные функции и методы, которые используются в различных частях приложения.

**Особенности проекта**

**Spring Boot**: Проект разработан с использованием Spring Boot, что обеспечивает простую настройку и управление приложением.

В проекте используется архитектурный подход MVC (Model-View-Controller), который обеспечивает разделение приложения на компоненты для управления данными, представления и бизнес-логики.

Для работы с базой данных используется JPA (Java Persistence API) с помощью Hibernate.

Авторизация и аутентификация пользователей реализованы с помощью JSON Web Token (JWT) и интеграции с Okta.

Приложение поддерживает механизм CORS (Cross-Origin Resource Sharing) для обеспечения безопасности и контроля доступа к ресурсам с разных источников.

Использование **`spring-boot-starter-data-rest`**: Проект использует **`spring-boot-starter-data-rest`**, что позволяет автоматически создавать RESTful API на основе JPA репозиториев. Это значительно упрощает создание и обслуживание конечных точек (endpoint'ов) API, а также обеспечивает их соответствие стандартам REST.

- **Пользовательские роли**: Пользователи и администраторы имеют различные права доступа к функциям приложения.
- **Система сообщений**: Пользователи могут общаться с администратором через систему сообщений, предоставляемую приложением.
